# Slot Scene (Pixi.js + TypeScript)

Тестовое задание: реализация прототипа слот-игры 5×3 с акцентом на архитектуру и предсказуемый поток данных.

## Запуск проекта

```bash
npm install
npm run dev

```

1. Архитектура

Проект реализован с разделением на независимые слои:

- Domain Model (GameModel)
  Содержит бизнес-логику
  Баланс, ставка, выигрыш
  Применение результата спина
  Не зависит от UI и Pixi
- ViewModel (GameViewModel)
  Единый источник состояния
  Связывает UI и сцену
  Обрабатывает пользовательские действия
  Предоставляет реактивные данные

- UI
  Отображает состояние из ViewModel
  Отправляет пользовательские намерения
  Не взаимодействует напрямую со сценой

- Pixi Scene (SlotScene)
  Отвечает за визуализацию
  Выполняет анимации
  Наблюдает ViewModel
  Не содержит бизнес-логики

- State Machine (SlotStateMachine)
  Управляет игровыми этапами:
  idle
  spinning
  showingWin
  Централизует переходы состояний

2. Поток данных
   UI → ViewModel → Scene → ViewModel → Domain
   UI инициирует действия
   Scene реагирует на состояние
   Domain изменяет данные
   ViewModel синхронизирует всё

3. Реализованный функционал
   - Игровое поле 5×3
   - Запуск спина
   - Остановка спина (ускорение)
   - Авто-режим (Auto x5)
   - Выбор ставки
   - Баланс / ставка / выигрыш
   - Мок результатов

4. Анимации
   Рандомная прокрутка символов
   Остановка барабанов по колонкам
   Scale-эффект при остановке
   Подсветка выигрышной линии

5. Ключевые решения
   -Избегает прямой связи между UI и сценой и дает предсказуемый поток данных

   -Централизует логику и нет разбросанной логики по коду

   -Бизнес-логика изолированаи это легко тестировать и переиспользовать

6. Структура проекта
   src/
   domain/
   viewModel/
   pixi/
   ui/
   mock/
   main.ts
   index.css

7. Возможные улучшения

8. Итог
